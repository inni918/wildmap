## 🔴 已修復的 Bug

### BUG-01 [P1 → 已修復] 車宿篩選顯示 0 個地點
- **原因**：Supabase PostgREST 預設上限 1000 筆，`.range(0, 9999)` 實際只返回前 1000 筆（全為露營），車宿 1,290 筆全部被截斷
- **修法**：改為分頁載入（PAGE_SIZE=1000 + `.order('id')`）循環讀取所有資料
- **影響**：車宿類別完全無法使用（P1）
- **Commit**：`fix: 修復車宿篩選 0 個地點的 P1 bug`
- **驗證**：修復後手機版確認顯示「3203 個地點」✅

### BUG-02 [P1 → 已修復] 搜尋框展開後被 Filter Bar 遮住
- **原因**：Header 搜尋框展開時高度增加（56px → 106px），但 Category Filter Bar 硬編碼 `top-14`（56px），直接覆蓋搜尋框
- **修法**：
  - Header.tsx：新增 `onSearchExpandedChange` prop 回報展開狀態
  - Map.tsx：追蹤 `searchExpanded` 狀態，動態調整 filter bar、FeatureFilter、列表模式的 top 值
  - FeatureFilter.tsx：新增 `searchExpanded` prop 動態調整位置
- **Commit**：`fix: 修復搜尋框展開時被 filter bar 遮住的 bug`
- **驗證**：搜尋「日月潭」→ 正確顯示 7 個地點，搜尋框可見 ✅

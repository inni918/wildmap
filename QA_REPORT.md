## 🔴 已修復的 Bug（累計）

### BUG-01 [P1 → 已修復] 車宿篩選顯示 0 個地點（第一輪）
- **原因**：Supabase PostgREST 預設上限 1000 筆，車宿 1,290 筆全部被截斷
- **修法**：改為分頁載入（PAGE_SIZE=1000）循環讀取所有資料
- **Commit**：`fix: 修復車宿篩選 0 個地點的 P1 bug`

### BUG-02 [P1 → 已修復] 搜尋框展開後被 Filter Bar 遮住（第一輪）
- **原因**：Header 搜尋框展開時高度增加，但 Filter Bar 硬編碼 top 值
- **修法**：動態調整 filter bar 的 top 值
- **Commit**：`fix: 修復搜尋框展開時被 filter bar 遮住的 bug`

### BUG-03 [P1 → 已修復] 個人頁面未登入時無限 spinner（第二輪新增）
- **功能**：個人頁面（/profile）
- **預期行為**：未登入用戶應看到「請先登入」提示或重導至登入頁
- **實際行為**：頁面顯示無限載入 spinner，等待 5 秒以上仍無反應
- **原因**：`authLoading` 為 `false` 且 `user` 為 `null` 時，`useEffect` 中有 `router.push('/login')` 但 render 函數返回 `null`（空白畫面），在 Supabase auth 初始化較慢時，spinner 會持續顯示
- **修法**：
  - 將 `if (!user) return null` 改為顯示友善的「請先登入」提示頁面
  - 包含登入按鈕（前往登入）和返回地圖連結
  - Loading 狀態也加入「載入中...」文字
- **嚴重程度**：P1（用戶體驗嚴重受損）
- **Commit**：`fix: 修復個人頁面未登入時無限 spinner 的 P1 bug`
- **已 push**：✅

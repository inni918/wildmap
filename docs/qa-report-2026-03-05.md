## ❌ 嚴重失敗項目

### Bug #1：登入後 spots 查詢回傳 0 筆（P0 — 嚴重）

**現象**：
- 未登入狀態：首頁正常顯示「5 個地點」
- 登入後：首頁顯示「0 個地點」
- 登出後清除 storage 再刷新：恢復「5 個地點」

**根因推測**：
Supabase `spots` 表的 RLS (Row Level Security) SELECT policy 對 `authenticated` role 的過濾邏輯有問題。可能只允許 `anon` 讀取，或 policy 中有條件過濾導致已登入用戶看不到任何 spots。

**重現步驟**：
1. 開 https://wildmap-dusky.vercel.app（未登入）→ 顯示 5 個地點
2. 登入任何帳號
3. 首頁顯示 0 個地點

**Console 相關錯誤**：
```
@supabase/gotrue-js: Lock "lock:sb-gwopbfqgtwrqgrhgpepx-auth-token" 
was not released within 5000ms.
```

**影響**：所有已登入用戶看不到任何地標 — **核心功能完全失效**

- 頭像字母取 Email 前綴首字母（Q）而非顯示名稱

**重現步驟**：

4. 點擊頭像 → 顯示 email 而非 display_name

## ⚠️ 注意事項

### 1. 登入 API 間歇性 CORS 錯誤
- 第一次登入嘗試出現 `Failed to fetch`（CORS 錯誤）
- 第二次嘗試成功
- Console 顯示 `Access-Control-Allow-Origin` 缺失
- 可能是 Supabase 冷啟動或暫時性網路問題

### 2. 錯誤訊息未中文化
- 登入錯誤顯示英文 `Invalid login credentials` 而非中文
- 忘記密碼的無效 Email 顯示英文 `Email address "..." is invalid`

### 3. Supabase Auth Lock 警告
- Console 持續出現 auth token lock 未釋放的警告
- 可能影響 session 管理穩定性

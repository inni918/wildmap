# 遊戲化 UI 設計研究報告
> Wildmap UX 研究 · 2026-03-04

---

## 概覽

本報告針對五大主流平台的遊戲化 UI 設計進行系統性分析，重點研究成就/徽章系統、進度條、等級視覺、排行榜及個人頁面呈現，並提供 Wildmap 成就系統的 UI 設計建議。

參考平台：Duolingo、Strava、Google Maps Local Guide、Stack Overflow、Steam

---

## 一、Duolingo

### 平台背景
全球最大語言學習 App，1.3 億月活躍用戶（2025 Q1）。遊戲化是 Duolingo 的核心競爭力，被業界公認為行動端遊戲化設計的標竿。

### 1. 成就／徽章視覺設計

- **連勝火焰（Streak）**：橘紅色火焰 emoji 搭配天數數字，是最醒目的視覺元素。顏色飽和度高，第一眼就看到。連勝天數越高，火焰動畫越劇烈。
- **聯賽徽章**：分為銅、銀、金、藍寶石、紅寶石、翡翠、紫水晶、珍珠、鑽石、黃金 10 個聯賽，每個聯賽對應不同寶石質地的盾牌圖標，設計精緻、擬物化。
- **成就系統（Achievements）**：圓形徽章為主，有「第一個朋友」、「連勝 7 天」等，採用扁平化插畫風格，角色（Duo 貓頭鷹）常出現在徽章圖案中，增加品牌一致性。
- **獲得動效**：徽章解鎖時有金色粒子爆炸 + 音效，強烈的即時回饋。

### 2. 進度條呈現

- **課程進度條**：綠色填滿條，搭配心形（生命值）系統。進度條在課程畫面頂部，清楚可見。
- **日常目標進度**：圓形進度圖，顯示「今日已賺 XP / 目標 XP」，達標後轉為金色慶祝動畫。
- **聯賽進度**：週排行榜 + XP 累積進度，頂端顯示「晉級區（綠色）」vs「降級區（紅色）」，製造緊張感。
- **技術細節**：進度條帶有彈跳動畫（spring animation），填充速度略帶延遲，讓玩家有「看到自己成長」的感受。

### 3. 等級系統視覺呈現

- **聯賽制度**（League System）是 Duolingo 的主要等級系統，每週根據 XP 點數競爭，排名高者晉升，排名低者降級。
- 各等級視覺差異明顯：銅牌（土黃色）→ 鑽石（藍白閃耀），越高階的視覺設計越精緻、越有光澤感。
- **課程等級**（Crown System）：每個技能可以升級 1-5 顆皇冠，皇冠圖示從空心到填滿，視覺清晰。

### 4. 排行榜設計

- 每週聯賽排行榜，顯示同聯賽 30 位用戶。
- 第 1 名顯示金色皇冠，前 3 名有特殊底色（金/銀/銅）。
- 自己的名次用藍色框標示，即使不在畫面內也會顯示在底部。
- 「晉級區」（前 5 名）用綠色漸層區塊標示，「降級區」（後 5 名）用紅色漸層，製造明顯的動機壓力。

### 5. 個人頁面成就展示

- 展示「連勝天數」、「本週 XP」、「本月 XP」、「總 XP」、「聯賽等級」。
- 徽章牆：可收集的成就徽章以網格排列，已解鎖的全色顯示，未解鎖的灰階半透明，製造「缺口效應」（Zeigarnik Effect）。
- 顯示「前 X% 學習者」的社會比較標籤，激勵持續學習。

### 6. 讓人「想收集」的關鍵設計

- 🔥 **連勝焦慮**：連勝天數讓用戶「不敢斷」，每天回來維持。
- 🏆 **聯賽制競爭**：每週都有重置，製造持續的競爭動機。
- 💎 **稀缺感**：鑽石聯賽是最高等級，到達即有身份感。
- 🎯 **近距原則**：進度條快滿時，特別想完成它（Goal Gradient Effect）。

## 二、Strava

### 平台背景
全球最大運動社群 App，追蹤跑步/騎車等戶外運動，以社群競爭為核心驅動力。

### 1. 成就／徽章視覺設計

- **Kudos（鼓勵）**：閃電圖示，類似 Facebook 按讚但更有能量感，橘色搭配。
- **成就徽章**：以「里程碑」為主（首次跑 5K、首次半馬等），設計偏簡約扁平，以運動圖標為中心，底色採用品牌橘色系。
- **KOM/QOM（King/Queen of the Mountain）**：區段最快紀錄標示，皇冠圖示，紅底白字，是 Strava 最具標誌性的視覺元素，也是最激發競爭心的設計。
- **個人紀錄（PR/CR）**：金色星形標記，出現在活動完成頁面。

### 2. 進度條呈現

- **年度目標**：用戶可自設跑步/騎車里程年度目標，首頁顯示環形進度圖，已完成百分比。
- **挑戰進度**：參加月度挑戰（如「11月跑 100 公里」），進度條設計清晰，搭配里程數和百分比。
- **Fitness & Freshness（付費功能）**：心肺能力曲線圖，以折線圖呈現「體能」vs「疲勞」。

### 3. 等級系統視覺呈現

- Strava 沒有傳統等級系統，但有隱性分層：
  - **KOM/QOM 持有者** → 頂端精英
  - **挑戰完成者** → 中階積極用戶
  - **一般追蹤者** → 基礎用戶
- 徽章式里程碑（Milestone Badges）作為非正式等級標記。
- 付費 Strava Summit 有額外功能，但無明顯等級視覺差異。

### 4. 排行榜設計

- **區段排行榜（Segment Leaderboard）**：每條路段都有排行榜，顯示 KOM 和前 10 名時間，個人最佳時間標示與差距。
- **好友排行榜**：在區段內只看好友排名，降低挫敗感，更容易超越。
- **挑戰排行榜**：月度挑戰中顯示好友間的完成進度比較。
- 排行榜使用橘色品牌色搭配深灰底，配色對比強烈，重點突出。

### 5. 個人頁面成就展示

- **統計數字**（跑步總里程、最長單次距離、最多爬升等）是主角。
- **年度報告（Year in Sport）**：年底自動生成個人年度回顧，類似 Spotify Wrapped，極高分享率。
- 挑戰徽章以方形磁鐵式設計排列，品牌贊助商設計精美徽章（Nike、GoPro 等），增加收集欲。
- 最近活動以地圖軌跡縮圖呈現，視覺豐富。

### 6. 讓人「想收集」的關鍵設計

- 👑 **KOM/QOM 競爭**：公開的最快紀錄，讓人想挑戰和保衛。
- 🤝 **社群壓力**：好友的 Kudos 和留言，社交認可是最強驅動力。
- 📅 **品牌聯名挑戰徽章**：Nike Run Club 月度挑戰、GoPro 挑戰等，徽章設計精美，有品牌加持的稀缺感。
- 📊 **數字成就感**：看到自己的里程數增長、個人紀錄被刷新，純數字就能帶來成就感。

## 三、Google Maps Local Guide

### 平台背景
Google 地圖的貢獻者計劃，鼓勵用戶提交評論、照片、地點資訊，以點數和等級制度激勵貢獻。

### 1. 成就／徽章視覺設計

- **Local Guide 徽標**：藍色六角形徽章，中間數字代表等級（1-10），設計簡潔有辨識度。
- 各等級徽章統一風格：藍色系，六角形輪廓，數字字體清晰。
- 無複雜插畫，強調「官方認證」的視覺語言，設計更接近職業認證而非遊戲。

### 2. 進度條呈現

- 個人 Local Guide 頁面顯示等級進度條：「目前點數 / 升下一級所需點數」。
- 貢獻統計（評論數、照片數、編輯數等）以數字列表呈現，缺乏動態進度條設計。
- 整體進度呈現較為樸素，不如 Duolingo 有視覺衝擊力。

### 3. 等級系統視覺呈現

- **10 個等級**：Level 1（0點）→ Level 10（100,000點以上）
- 等級數字直接顯示在徽章上，清晰直觀。
- 高等級（7-10）有「專家」地位的隱性社群認可。
- 高等級用戶在評論下方有特殊標示，增加社群可信度。

### 4. 排行榜設計

- **Google Maps Local Guide 沒有公開的全球排行榜**，這是刻意的設計選擇，避免刷分行為。
- 強調「累積貢獻」而非「競爭排名」，屬於合作型遊戲化而非競爭型。
- 社群活動（Local Guide Connect）提供地區社群互動，但無正式排名系統。

### 5. 個人頁面成就展示

- 個人 Local Guide 頁面顯示：等級徽章、總點數、各類貢獻統計（評論、照片、回答、地點等）。
- 點數來源分類呈現（哪些行為得了多少點），有助於引導用戶行為。
- 限時「徽章成就」（如特定挑戰期間完成任務）偶爾發放，增加稀缺感。

### 6. 讓人「想收集」的關鍵設計

- 🔵 **身份認同**：Level 7+ 被稱為「在地嚮導專家」，有真實的社群地位。
- 📸 **多元貢獻管道**：評論、照片、影片、問答、地點建議，降低參與門檻。
- 🎁 **實質獎勵**：高等級 Local Guide 曾受邀參加 Google 舉辦的實體活動，有真實利益驅動。
- 🗺️ **地圖貢獻的永久性**：看到自己新增的地點出現在 Google 地圖上，有真實的成就感。

## 四、Stack Overflow

### 平台背景
全球最大程式設計問答社群，2008年創立，現有 2900萬+ 用戶。聲望和徽章系統是其核心機制，也是學術界研究遊戲化的重要案例。

### 1. 成就／徽章視覺設計

- **三級徽章制度**：
  - 🥉 **銅牌（Bronze）**：棕色圓點，基礎成就
  - 🥈 **銀牌（Silver）**：銀色圓點，中階成就
  - 🥇 **金牌（Gold）**：金色圓點，頂級成就（極稀有）
- 徽章圖示極度簡化：只有一個實心圓點 + 顏色區分，強調「稀缺性」而非「美觀性」。
- 金牌徽章數量極少（如「Great Answer」需要 100 個讚），稀缺性本身就是設計。
- 徽章名稱富有文化意涵（Fanatic = 連續 100 天登入、Legendary = 極高聲望），命名策略強化身份認同。

### 2. 進度條呈現

- 聲望（Reputation）以數字呈現，無傳統進度條。
- 個人頁面有「聲望圖表」（折線圖），顯示聲望隨時間的增長趨勢，滿足「看到成長」的心理需求。
- 徽章取得進度在「下一個徽章」提示中顯示（如「你還差 3 個讚即可獲得 Nice Answer」）。

### 3. 等級系統視覺呈現

- Stack Overflow 以**聲望數值**而非等級制度呈現資歷：
  - 1 → 新手（可發問）
  - 15 → 可投票
  - 50 → 可留言
  - 2,000 → 可編輯他人貼文
  - 10,000 → 可刪除貼文
  - 聲望解鎖功能，而非虛榮視覺，功能權限才是「等級」的實質意義。
- 用戶名旁的聲望數字是唯一視覺標記，設計極簡但信息量豐富。

### 4. 排行榜設計

- **全站排行榜**：按聲望排名，可過濾週/月/年/全期間。
- **標籤排行榜**：在特定技術標籤（如 Python、JavaScript）中的排名，更有針對性。
- 排行榜設計非常樸素（純文字列表），但對開發者群體有極強吸引力，因為高排名代表真實的技術認可。
- 個人名旁顯示國家旗幟，增加地區競爭感。

### 5. 個人頁面成就展示

- 顯示：聲望數字（最醒目）、金/銀/銅徽章數量、近期活動。
- 徽章列表以三欄排列（金/銀/銅），已獲得的徽章全彩顯示。
- **技能標籤 + 分數**：每個回答的技術標籤都累計得分，個人頁顯示「Top 5% Python 回答者」等排名標籤，極具說服力。

### 6. 讓人「想收集」的關鍵設計

- 💡 **功能性獎勵**：聲望不只是數字，真實解鎖網站功能，有實用動機。
- 🔑 **金牌的極稀缺性**：很多用戶用了十幾年只有個位數金牌，稀缺性讓每個金牌都珍貴。
- 🌐 **全球技術社群認可**：高聲望等於公開的技術履歷，對求職有實質影響。
- 📌 **「Fanatic」挑戰**：連續 100 天登入，製造習慣養成的強迫循環。

## 五、Steam

### 平台背景
Valve 的全球最大 PC 遊戲平台，成就系統是其最核心的遊戲化元素之一，也是整個業界學習的對象。

### 1. 成就／徽章視覺設計

- **遊戲內成就（Achievements）**：每款遊戲自行設計，風格多樣，但統一顯示於 Steam 個人頁。
- **獲得時圖示**（彩色）vs **未獲得圖示**（灰階），對比鮮明，製造「缺口感」。
- **稀有度標示**：每個成就旁顯示「全球 X% 玩家解鎖」，稀有成就（<5%）讓人特別有炫耀欲。
- **遊戲徽章（Trading Cards）**：蒐集遊戲卡牌製作徽章，有 1-5 個等級，每個等級徽章設計不同（更精緻、有光效），可展示在個人頁。

### 2. 進度條呈現

- **成就完成度**：個人成就頁顯示「已解鎖 X / Y 個成就（Z%）」附帶進度條。
- **徽章升級進度**：集齊一套卡牌可製作徽章，需要重複蒐集卡牌才能升級，製造重複驅動。
- **Steam 點數商店**（Points Shop）：消費 Steam 時獲得點數，點數用於購買 Profile 裝飾品，有「點數餘額」顯示。

### 3. 等級系統視覺呈現

- **Steam 等級（Steam Level）**：以數字等級呈現，通過購買遊戲、蒐集徽章、參與活動獲得 XP。
- 等級數字旁有彩色邊框，高等級（50+、100+）有特殊邊框顏色（金色、彩虹）。
- 等級影響好友上限和市集功能，有功能性意義。
- 高等級玩家（Level 100+）在社群中有明顯的展示欲。

### 4. 排行榜設計

- **遊戲內排行榜**：個別遊戲的得分/通關時間排行，全球/好友分開顯示。
- **Steam 等級好友排行**：可查看自己與好友的 Steam 等級排名。
- 沒有全平台統一的公開排行榜，各遊戲自行設計。
- 排行榜視覺設計偏向工具感（數字列表為主），但在遊戲主頁有視覺化展示。

### 5. 個人頁面成就展示

- Steam 個人頁（Profile）高度可自訂：
  - **Featured Achievements**：手動置頂的成就展示，可選罕見成就炫耀。
  - **遊戲徽章**：展示已製作的精美徽章（可設計個人 Showcase）。
  - **成就展示櫃（Achievement Showcase）**：展示稀有度最高的成就，旁邊標示「全球 X% 解鎖」。
  - **完美主義者標記**：100% 完成所有成就的遊戲有特別標示，是硬核玩家的榮譽。
- Profile 可以用 Steam 點數購買動態背景、貼圖框、頭像框，個性化程度極高。

### 6. 讓人「想收集」的關鍵設計

- 📊 **稀有度百分比**：「全球 0.8% 玩家擁有」是最強的收集驅動力，稀缺感直觀。
- 🃏 **卡牌蒐集機制**：將成就與卡牌交易市場結合，有市場經濟邏輯，增加收集的複雜樂趣。
- 🎨 **高度個性化 Profile**：把個人頁當作「展覽館」，讓收集行為有了公開展示的舞台。
- ✅ **完美主義驅動**：100% 成就完成帶來的完結感（Completion Itch），是最強的長期留存機制。

## 六、跨平台設計模式總結

### 「想收集」的六大心理機制

| 機制 | 說明 | 代表平台 |
|------|------|----------|
| **缺口效應** | 看到未完成的灰色圖示，強迫症想填滿 | Steam、Duolingo |
| **稀缺感** | 「全球 X% 擁有」讓成就有身份感 | Steam、Stack Overflow |
| **社群認可** | 公開排行/好友看到，社交壓力驅動 | Strava、Stack Overflow |
| **功能性獎勵** | 解鎖真實功能而非虛榮，更有動機 | Stack Overflow、Google Maps |
| **進度可見性** | 看到自己的成長進度，想繼續推進 | Duolingo、Strava |
| **習慣養成** | 連勝/連登入機制，讓用戶不敢停止 | Duolingo、Stack Overflow |

### 視覺設計共同特徵

1. **三層徽章制**（銅/銀/金）幾乎是全球標準，直觀易懂
2. **灰階 vs 彩色**對比呈現已得/未得成就
3. **動效即時回饋**：解鎖瞬間的動畫和音效不可省略
4. **數字具體化**：里程數、天數、聲望值，具體數字比「等級 5」更有成就感
5. **社交可見性**：成就要讓別人看到才有效，公開展示設計至關重要

## 七、Wildmap 成就系統 UI 建議

### 7.1 整體設計哲學

Wildmap 是一個**戶外探索平台**，成就系統應呼應「探索」、「自然」、「發現」的主題，視覺語言參考 FF14/魔獸世界的精緻感，同時保持行動端的易用性。

**設計關鍵字**：探索者、發現者、守護者、自然元素（山、水、森林、星空）

---

### 7.2 徽章系統設計

#### 徽章分級（建議五級制）

| 等級 | 名稱 | 視覺 | 稀有度 |
|------|------|------|--------|
| ⚪ | **旅人** | 銀色邊框、輕量線條 | 常見，引導入門 |
| 🟢 | **探索者** | 翠綠色邊框、立體感 | 進階行為 |
| 🔵 | **嚮導** | 藍寶石邊框、光澤感 | 持續貢獻者 |
| 🟣 | **守護者** | 紫色深邃邊框、特效 | 資深核心用戶 |
| 🟡 | **傳奇旅人** | 金色閃耀邊框、動態光效 | 極稀有，<1% 玩家 |

#### 徽章視覺設計原則

1. **主題化插畫**：每個徽章有獨立的自然主題插畫（百岳圖示、海邊日落、雨林植物、星空等），而非抽象符號。
2. **等級升級視覺差異**：同一成就的不同等級，插畫細節逐漸增加（如「健行者」徽章：初級=簡單山形，傳奇=細節豐富的百岳全景）。
3. **光效分層**：傳奇/守護者等級徽章加入 CSS shimmer 動效或 Lottie 動畫，讓高階徽章「活起來」。
4. **灰階未解鎖設計**：未解鎖徽章以剪影/灰階顯示，並附上神秘提示（「在 X 地點完成某行為即可解鎖」），製造期待感。

#### 建議徽章類別

```
🏔️  探索成就     首次踏上某地形（山地、海岸、森林、高原）
📸  記錄成就     上傳照片、撰寫評論、新增地點
🤝  社群成就     獲得他人點讚、被引用為推薦路線
⏱️  挑戰成就     特定時間內完成路線、參與活動
🗺️  收集成就     完成特定區域的所有打卡點
🌙  限定成就     節慶活動、季節性路線（稀缺設計）
```

### 7.3 進度條設計

#### 主要進度條形式

1. **探索進度環**（首頁主視覺）
   - 圓形進度環，顯示「本月探索目標完成度」
   - 顏色採用台灣山林的漸層色（山藍→森林綠）
   - 達標時觸發爆炸粒子動效

2. **區域收集進度條**（地圖頁）
   - 橫向進度條，顯示「縣市完成 X/Y 個打卡點」
   - 在地圖上覆蓋顯示，讓地理位置和進度結合
   - 完成度高的區域在地圖上有視覺標記（如熱力圖）

3. **成就解鎖進度提示**
   - 在行動後顯示「你還差 2 個評論即可解鎖『評論達人』徽章」
   - 使用 Toast 通知形式，非侵入式但有即時反饋

#### 進度條動效規範

- **彈跳填充**（spring easing）：不要線性，要有彈性
- **填充速度**：比實際速度略慢（0.5-1秒延遲），讓用戶「享受」進度填滿的過程
- **聲音設計**：進度完成時有輕微音效（可關閉），徽章解鎖有明顯音效

### 7.4 等級系統設計

#### 建議的探索者等級制度

```
Lv.1  🌱  新鮮人旅人    （0-99 EXP）
Lv.2  🥾  初階健行者    （100-499 EXP）
Lv.3  🗺️  熟悉嚮導     （500-1,999 EXP）
Lv.4  ⛰️  資深探索者   （2,000-9,999 EXP）
Lv.5  🏔️  百岳獵人     （10,000-49,999 EXP）
Lv.6  🌟  傳說旅人     （50,000+ EXP）
```

#### 等級視覺設計

- 等級數字旁有對應的自然主題邊框（非幾何邊框，而是樹葉/山形輪廓邊框）
- 個人頭像旁顯示等級標示，在評論/貼文中永遠可見（類似 Stack Overflow 的聲望）
- 等級提升動效：整個畫面的「等級提升」特效（ Level Up！），參考 RPG 遊戲體驗

#### EXP 獲得方式設計

| 行動 | EXP | 設計目的 |
|------|-----|----------|
| 首次打卡新地點 | +50 | 探索新地點 |
| 上傳照片 | +10 | 增加內容 |
| 撰寫評論 | +20 | 提升內容質量 |
| 獲得其他用戶的 👍 | +5 | 鼓勵高品質貢獻 |
| 完成月度挑戰 | +100 | 長期留存 |
| 首次登入（連續天） | +10/天 | 習慣養成 |

### 7.5 排行榜設計

#### 雙軌排行榜策略

**全國排行榜**（適度競爭）
- 按月度重置，降低追趕難度
- 顯示前 50 名，自己排名固定在底部
- 分為「新手組（Lv.1-2）」和「老鳥組（Lv.3+）」，避免新手挫敗感

**在地排行榜**（強化社群感）
- 縣市/鄉鎮層級的排行榜
- 「桃園市最活躍探索者」這類在地認同感更強
- 地圖上直接標示當地排名前三的用戶

**好友排行榜**（最溫暖的競爭）
- 只看好友間的排名，降低壓力
- 好友互動（Kudos 功能）整合在排行榜頁面

#### 排行榜視覺設計

- 第 1-3 名：金/銀/銅框 + 特殊徽章樣式展示
- 自己的名次：用品牌主色（山藍色）框線標示
- 排名變動：顯示「↑ 3」（綠色上升）或「↓ 2」（紅色下降）
- 排行榜頁背景：淡化的山景插畫，強化主題感

### 7.6 個人頁面成就展示

#### 個人頁布局建議

```
┌─────────────────────────────────┐
│  頭像 + 等級邊框 + 暱稱           │
│  Lv.4 資深探索者 | EXP: 3,456     │
├─────────────────────────────────┤
│  ⭐ 核心成就展示（3個精選徽章）     │
│  [山林探索者] [百日旅人] [攝影達人]│
├─────────────────────────────────┤
│  📊 探索統計                      │
│  打卡 47 地點 | 評論 23 則 | 照片 89 │
├─────────────────────────────────┤
│  🗺️ 足跡地圖（mini 版）           │
│  已探索的縣市有顏色標記            │
├─────────────────────────────────┤
│  🏆 徽章收藏（可展開）            │
│  旅人(3) 探索者(5) 嚮導(2) 守護者(0)│
└─────────────────────────────────┘
```

#### 「足跡地圖」是 Wildmap 的殺手級設計

- 個人頁顯示台灣地圖，已打卡的縣市/鄉鎮有顏色填滿
- 顏色深度對應打卡次數（像熱力圖，去越多次越深色）
- 「台灣探索度：47%」這類整體進度數字，製造強烈的完成欲
- 可分享到社群媒體（「我探索了台灣 47% 的土地！」），病毒傳播設計
